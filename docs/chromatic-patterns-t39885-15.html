Impossibility argument:
123 diagonals in b124 must all have the same parity.
b24 cannot have the same permutation, as that would eliminate the last digit out of the triple in b5.
Therefore they have different permutations with the same parity, so the digit in r4c2 cannot appear in r2c4.
The digit in r4c2 is in r2 forced into r2c89, which eliminates it out of the triple in c7, ie. contra.

Hi eleven,

The provided tables aren't an output of checker. They are hardcoded patterns that checker uses for UA searching in a given grid.
Short UA are covered - short in sense number of cells occupied, not the values involved. Ed Russel is attributed as a contributor for the larger patterns, and the list is incomplete.

I am not sure if in this context impossible pattern is equivalent to unavoidable set.

Considering 3-digit UA it should be possible to generate all of them. From this post you have to deal with 92048 essentially different 3-rookeries.
All you have to do is for each rookery to take one of its full solution grids, to clear the cells covered by the rookery, to find and compare all solutions to each other, to exclude from the so found UA those that are not minimal (i.e. are superset of other UA in the same list), and finally to canonicalize and remove duplicates.
This will give you also all 2-digit UA. If you keep a track of the possible digits within each cell, it would be easier to remove 2-digit UA. Also I am expecting some cells to contain only 2 interchangeble digits instead of 3, which you can exploit later since it is an additional restriction.
