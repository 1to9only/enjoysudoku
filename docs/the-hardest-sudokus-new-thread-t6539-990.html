The majority of the new ones are from the expander script - taking an 11.7 (or 11.6), adding all possible singles, and then finding all minimals. These puzzles are guaranteed to have singles available as a result. Any given hard puzzle with singles may come with a large number of friends (whereas when it does find a puzzle with no singles, there is nothing to expand, so those usually come alone or with a small number of close neighbors).

The number of puzzles is a bit misleading, of course, since puzzles from the same non-minimal are (usually) the same in some sense. At some point I may add a column to the database for this (the minlex form of the non-minimal parent). (Occasionally a minimal will be harder than the parent, removing a digit that can't be placed as a single, but it's pretty rare.)

(I do have an 11.6 pearl in the currently running batch, at 23c. The rest are /1.2/1.2 though.)
